# Serverless HTTP 服务

Serverless HTTP 服务基于腾讯云 API 网关 和 Web Cloud Function（以下简称“Web Function”）建站云函数（云函数的一种类型）的产品能力，可以支持各种类型的 HTTP 服务开发，实现了 Serverless 与 Web 服务最优雅的结合。用户可以快速构建 Web 原生框架，把本地的 Express、Koa、Nextjs、Nuxtjs 等框架项目快速迁移到云端，同时也支持 Wordpress、Discuz Q 等现有应用模版一键快速创建。

### 简化配置

Serverless HTTP 方案无需复杂配置，通过部署 Web Function，即可快速、高效构建您的 RESTful API，基于云函数提供的多项高级能力，助力业务上线。

![](https://qcloudimg.tencent-cloud.cn/trisys/assets/product/images/qLmxYbr7QrdEyV4dizz-h.png)

### 平滑迁移

Web Function 支持接收与处理原生 HTTP 请求，无需再做事件格式转换，原生 Web 框架仅需几行代码，即可平滑迁移上云。

![](https://qcloudimg.tencent-cloud.cn/trisys/assets/product/images/erHqQRo0o59_9U3xpcxSp.png)

### 多语言支持

Serverless HTTP 提供了不同开发语言常见 Web 框架的部署方案，满足不同业务场景和开发习惯。

![](https://qcloudimg.tencent-cloud.cn/trisys/assets/product/images/QEebIvAwnvMagwAHquuQC.png)

### 标准化

完美支持 Swagger/OpenAPI 规范，构建标准化、统一的 API 平台，保证业务间的便捷通信。

### 弹性扩展

Serverless HTTP 基于API 网关和 Web Function（建站云函数），支持根据流量的波峰波谷自动扩缩容，减少手动配置，降低运维复杂度，节省使用成本。

### 安全可靠

通过 SSL 加密，保证 API 的安全通讯；通过可靠的鉴权方式，实现对来源访问权限的 token 认证，保证 API 及业务的安全性。

### Serverless HTTP API 支持了哪些Web 应用框架？

目前 Serverless HTTP API 通过 Component 支持多种开发语言的 Web 框架，如 Node.js Koa、Express、React、Vue 等，支持 SSR 相关的 Next.js 和 Nuxt.js 框架；Python 支持 WSGI 框架如 Flask、Django 等；PHP 支持框架如 Laravel、ThinkPHP 等。更多贴合实际应用场景的框架在持续输出中。
