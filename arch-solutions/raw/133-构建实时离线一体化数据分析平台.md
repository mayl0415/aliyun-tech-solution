## 多源汇集 AnalyticDB MySQL，搭建一站式大数据分析平台

支持多数据源联合分析，解决跨多实例数据库关联和分库分表所带来的复杂问题，让分析师和数据工程师更专注于业务逻辑。通过物化视图构建轻量级 ETL 分析任务，省去复杂的数据任务编排，节省调度资源。同时具备在线分析和 ETL 计算一体化，实现大数据与数据库的融合，通过资源组隔离保证业务稳定运行。

**在离线一体化**

支持数据实时增删改、具备在线分析和ETL 计算一体化，实现大数据与数据库的融合，通过资源组隔离保证业务稳定运行。

**简化 ETL 流程**

支持通过物化视图将用户定义的查询提前计算好，按用户要求自动刷新视图中的数据，加速复杂查询和简化 ETL 流程。

**多源联合分析**

支持多数据源联合分析，解决跨多实例数据库关联和分库分表所带来的复杂问题，让用户可以更专注于业务逻辑。

## 云原生数据仓库AnalyticDB MySQL 实现数据一站式分析与洞察

![](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/1539347371/p907327.png)

本方案通过 DTS（数据传输服务）实现 MySQL 和云原生数据库 PolarDB 到云原生数据仓库 AnalyticDB MySQL 版的实时数据同步。DTS 提供全量校验和增量校验功能，确保数据同步的源端和目标端之间的数据一致性。在云原生数据仓库 AnalyticDB MySQL 版中创建 Job 型资源组 应用于高吞吐离线跑批场景、交互式资源组（ Interactive 资源组）应用于在线场景或者交互式分析场景，最终由云原生数据仓库 AnalyticDB MySQL 版对外提供应用查询服务。

  结合数仓分层设计为大屏展示提供实时和离线数据支持，满足实时监控和决策需求。
