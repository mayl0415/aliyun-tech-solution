## 网站业务在提升访问体验面临的挑战

互联网业务通常要求具备高并发处理能力和全球化低延迟访问，对于访问体验以及数据安全具有较为严苛的要求，传统的方案是使用 CDN 对页面中的静态资源进行加速访问以提升页面的访问体验，在页面响应、系统稳定、数据安全等方面还是会面临一定的挑战。

**访问延时高**

客户端遍布全球，需实时回源站获取数据，全球公网链路复杂，回源效果无法保证，用户访问性能不稳定，页面加载缓慢。

**核心数据易遭受攻击**

恶意盗刷、大流量攻击、内容盗取等安全攻击经常发生，导致回源慢、响应慢，影响终端用户访问体验，造成用户流失。

**配置复杂、难管理**

不同加速域名和回源条件，需要更为灵活的自定义规则满足加速要求；多个加速域名间存在配置相同的域名，需要批量管理。

## 传统 CDN 加速和基于 ESA 加速对比

阿里云边缘安全加速 ESA 是一个由遍布全球各地的基础节点设施组成的全球化网络平台。面对复杂的全球网络环境和无处不在的网络攻击威胁，能够为您的业务（包括但不限于游戏、电商、金融、零售行业）提供一站式的全面加速、安全保护和就近计算等服务。

| **CDN 加速方案** | VS | **ESA 加速方案** |
| --- | --- | --- |
| - **非缓存类业务加速效果一般**  传统 CDN 主要针对图片、视频等静态内容进行缓存和加速，对于动态内容例如 API 或需要回源的请求，加速质量一般，偶尔出现响应时间较长的情况。 | 性能 | - **更好的加速体验**  ESA 的缓存加速效果对比 CDN 毫不逊色，对于非缓存类业务的加速性能更上一层楼；ESA 基于智能路由、全球网络实时感知、拥塞算法升级，使 Web 应用性能平均提升 30%；ESA 支持缓存规则、重定向规则、免费证书、源站证书校验、即时日志、等候室、版本管理等更高阶的流量管理能力，快速自定义加速规则，帮助运维排查问题，提升管理效率。 |
| - **不支持边缘计算**  不支持在边缘节点部署代码和运行轻量级服务，无法实现就近计算和存储，响应速度和动态内容处理受限。 | 灵活性 | - **易上手的边缘计算**  基于 Serverless 架构的服务允许开发者编写 JavaScript 代码并在阿里云全球边缘节点上部署和执行，支持 ES6 语法和标准的 Web Service Worker API，支持通过边缘容器部署业务镜像下沉算力，用户的请求可以直接在就近边缘节点上得到响应处理，从而显著减少延迟、提高响应速度，实现更低时延的计算体验。 |
| - **安全防护能力一般**  仅支持 Referer 防盗链、URL 鉴权、IP 黑白/名单等基础安全防护机制，缺乏 WAF，DDoS 等高阶防护机制，导致恶意流量回源至源站引发安全故障。 | 安全性 | - **全面的安全防护能力**  在边缘节点集成原生的 WAF、DDoS 的防护能力，通过 Anycast 就近防护智能清洗，将大流量攻击阻断在边缘；可以设定 WAF、Bot 规则，将应用层攻击拦截，防止接口被爆破、流量被刷等恶意攻击，使数字中心远离安全攻击风险，为终端用户提供稳定的访问服务。 |
| - **计费项多、对账较复杂**  CDN 的计费逻辑为按量付费，若开通的功能较多，会产生多个计费项的计量出账，需随时关注账号的信控等信息，每个月对账也增加了复杂度。 | 成本 | - **计费规则简单易懂**  ESA 为客户提供基础版、标准版、高级版三个固定套餐，客户可以根据自身业务特点按需选择，套餐内仅按流量计费（请求数不计费），套餐外功能按需开启按量付费（如智能路由、边缘函数等）；为企业客户提供一客一议的定制套餐，若固定的三个套餐不能满足加速和安全要求，可以联系阿里云商务定制企业版。 |
