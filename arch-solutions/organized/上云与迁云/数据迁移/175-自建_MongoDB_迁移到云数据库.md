## ECS 自建 MongoDB 迁移云数据库前后对比

| **ECS 自建 MongoDB** | VS | **云数据库 MongoDB** |
| --- | --- | --- |
| - **短期省成本，长期反增负**  公网流量收费；备份数据占用空间；无现成数据库代理用于读写分离；无现成 Serverless 方案，扩容需额外采购。 | 成本 | - **低成本投入，高效率产出**  公网流量不收费；提供最多 2 倍于存储空间的免费备份空间；通用型数据库代理不收费；支持即时开通，弹性扩容。 |
| - **运维压力大，性能难调优**  从基础监控到性能调优、备份恢复和安全防护，均需依赖人工介入，运维复杂度高。 | 运维 | - **运维自动化，响应快速化**  借助 CloudDBA 智能诊断能力，系统可自动识别并分析数据库运行中的问题，提供针对性的优化建议，减少对传统人工运维的依赖，帮助用户以更低的成本实现更高效的数据库管理。 |
| - **高可用缺失，稳定性堪忧**  缺乏完善的高可用方案，需依赖专业人员手动搭建，实现复杂且技术门槛较高。 | 稳定 | - **服务高可用，数据高可靠**  支持同城单节点或三可用区的高可用性容灾方案，并能构建异地容灾实例。通过自动故障转移和数据冗余机制，确保在节点故障时系统仍能持续运行且数据不会丢失。 |
| - **传输不加密，漏洞靠人工**  无现成的连接链路加密；无现成的 TDE 加密；无 SQL 审计；内核 Bug 需要自行修复 | 安全 | - **体系化防御，各环节守护**  提供完善的安全防护能力，涵盖 DDoS 防护、漏洞自动修复、访问控制、网络隔离、数据加密、审计日志等功能，并由阿里云持续维护数据库内核安全，无需用户自行处理。 |

## 借助数据传输服务快速实现 ECS 自建 MongoDB 迁移到云数据库

![](https://help-static-aliyun-doc.aliyuncs.com/assets/img/zh-CN/6471230571/p974192.png)

本方案旨在通过数据传输服务（DTS），将部署在 ECS 上的 MongoDB 数据库迁移至云数据库 MongoDB。DTS 还提供全量与增量数据校验功能，能确保迁移过程中的数据一致性。借助云数据库提供的数据库管理服务（DMS）可视化操作界面，可进一步提升日常运维的效率与便捷性。

**部署时长：**30 分钟

**预估费用：**5 元（云数据库 MongoDB，ECS 实例，数据传输服务都是按量付费。假设您选择方案推荐的资源，且运行时间不超过 1 小时，预计费用 5 元左右。如果调整了资源规格，请以控制台显示的实际报价以及最终账单为准。）
