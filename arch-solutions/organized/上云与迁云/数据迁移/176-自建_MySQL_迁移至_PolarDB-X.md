## 从性能瓶颈到架构升级，PolarDB-X 助力数据库平滑演进

PolarDB 分布式版是阿里云自主设计研发的高性能云原生分布式数据库产品，通过了中国信息安全测评中心组织的测评，新增海光芯片支持，标准版 100% 兼容 MySQL，为用户提供高吞吐、大存储、低延时、易扩展和超高可用的云时代数据库服务。

**自建 MySQL 高可用能力不足**

自建 MySQL 高可用架构业界主流仍然采用主从复制 (Master-Slave)，数据库无法自感知数据节点故障，依赖于第三方组件或业务多次交互与探查，数据库持续服务性能力不足。

**自建 MySQL 弹性扩展能力不足**

自建 MySQL 多为单机架构，存储容量难以突破 3T 的限制，在高并发场景下易出现性能瓶颈，响应延迟高。扩容依赖手动分库分表，流程复杂且影响服务稳定性，难以灵活适配业务增长和流量波动。

**PolarDB-X 破解数据库性能瓶颈**

PolarDB-X 基于云原生分布式架构，支持自动扩展与多副本强一致性同步，提供 RTO<8S 的高可用保障，服务持续稳定；兼容 MySQL 协议，应用无需改造即可无缝接入，全面提升数据库服务能力。

## 阿里云 PolarDB-X 方案对比自建 MySQL 方案

| **自建 MySQL** | VS | **阿里云 PolarDB-X** |

| - **可用性弱**  配置和管理较复杂，主从异步复制，故障切换依赖于业务判断，数据库无法自感知，不可控因素较多；恢复流程复杂，业务连续性难以保障。 | 高可用 | - **可用性强**  默认同城三机房部署，内核协议级自动故障节点检测，发现主节点故障自动进行节点切换，业务连接会话保持，降低业务影响，持续提供数据库服务。 |
