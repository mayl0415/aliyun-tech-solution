## 产品简介

云数据库 Tair 是一种全托管、兼容 Redis 协议且具备超高性能的数据库服务，能够保证亚毫秒级的稳定时延，为应用程序起到加速作用。云数据库 Tair 包括 Redis 开源版内核和 Tair（又称为企业版）内核，您可根据需要进行选择。其中，Redis 开源版内核基于开源代码进行了强化，而 Tair 则在此基础之上进一步增加了大量的企业级特性，能够覆盖 Redis 开源版难以应对的场景，提供稳定可靠的服务。阿里云 Redis 开源版及 Tair 均提供了多种架构，并支持资源规格的灵活配置和实时调整，能够在游戏、金融、广告、电信、直播、电商、安全等对时延有严苛要求的领域提供稳定支撑。 

## 产品优势

兼容开源 Redis，在可靠性、稳定性、弹性、安全、易用性等多个维度进行增强，兼顾高性能缓存与持久化 KV 数据库的场景需求。
云数据库 Tair 让您无需关注故障恢复、软件更新、监控及备份管理等常见的运维和管理任务。Tair 可全自动地对实例进行监控管理，从而保证服务的连续性，同时不定期地对软件进行更新，持续增强产品的功能、性能、安全和稳定性。此外，Tair 内置多指标监控、大/热 KEY 分析、慢日志及时延洞察等观测性能力，能够在发生异常时协助您快速定位。
### 内核深度优化和高可用配置，确保服务始终在线
通过增强型内核保证亚毫秒级的低时延，还能够快速发现异常并完成切换；支持精确探活，可杜绝因请求异常造成的不合理切换；支持单 AZ、双 AZ、3 Region 多活三种高可用配置，可满足可用区或地域级的容灾需要。
### 支持多种架构之间在线转换，随时按需调整
支持高可用、智能读写分离、分布式（集群）、单节点四种架构，并支持不同架构之间不停机转换，您可根据需要随时调整；分布式（集群）架构最大支持 256 分片，能够完成高达 7000 万次/秒的低时延请求处理；接入智能读写分离架构无需修改任何代码，最大可提供 9 个只读节点支撑读负载。
### 多种存储介质按需选择，降本增效
Tair 内核支持三种不同的存储介质：内存、持久内存和磁盘（包括 ESSD 和 SSD）。根据应用程序对数据库性能、数据存储规模和资源成本的综合考量，您可以选择最适合的存储介质，从而节省 30%~70% 的数据存储成本。持久内存或磁盘与 Tair 内核搭配，不仅可以作为持久化的数据库，而且存储成本远低于内存；同时，您可以基于 Tair 的持久化特性对数据库架构进行简化，仅用 Tair 即可替代同时使用缓存和持久化数据库的架构，不仅可以进一步降低数据库的使用成本，还能提升应用程序的开发效率。

## 安全合规

云数据库 Tair 从网络、存储、备份、容灾、审计等方面，提供全面的安全加固功能，保障您的数据安全。
默认情况下，云数据库 Tair 不允许任何内网或公网 IP 地址访问，您需要将客户端的 IP 地址或IP地址段添加到 Tair 实例的白名单中，才能通过该客户端访问 Tair 实例，实现从访问源进行风险控制。
VPC（Virtual Private Cloud）是私有网络环境，通过底层网络协议，在网络二层完成网络隔离，具备安全可靠、灵活可控、简单易用的特性和较强的可扩展性。
云数据库 Tair 支持 TLS（Transport Layer Security）协议，该协议已成为互联网加密通信的标准协议，是 SSL（Secure Sockets Layer）协议的升级版，具有比 SSL 协议更好的加密技术和更高级别的安全性，可进一步保障数据通信安全。
云数据库 Tair 支持透明数据加密 TDE（Transparent Data Encryption），该功能可基于用户密钥对 RDB 数据文件执行加密和解密，提高数据的安全性。
云数据库 Tair 支持按备份集恢复数据和按时间点恢复数据，可最大限度地避免误操作带来的数据损失。
当云数据库 Tair 监测到实例的主节点不可用时，会自动触发主备切换，将备节点提升为主节点，保障实例的可用性。
云数据库 Tair 支持双机房的同城容灾架构，主备机房数据通过专门的复制通道同步，当主机房出现电力或网络问题时，备实例升级为主实例，保障服务可用。
云数据库 Tair 支持异地多个站点同时对外提供服务，提供跨域复制的能力，快速实现数据异地灾备和多活。
